# 部署前可删除文件清单

## 一、建议删除（上传前可删，服务器会重新生成）

| 路径 | 说明 | 约节省 |
|------|------|--------|
| `backend/node_modules/` | 依赖包，Docker 构建时会 `npm install` 重新安装 | ~200MB+ |
| `backend/dist/` | 后端构建产物，Docker 会重新 `npm run build` | ~几 MB |
| `frontend/node_modules/` | 前端依赖，Docker 构建时会重新安装 | ~150MB+ |
| `frontend/dist/` | 前端构建产物（如有），Docker 会重新构建 | ~几 MB |

**删除后**：用 Xftp 上传会快很多，Docker 部署时会在容器内重新安装和构建。

---

## 二、可选删除（视情况）

| 路径 | 说明 |
|------|------|
| `server_asset_backup.sql` | 根目录的数据库备份（约 3.2MB），部署不需要；**若需恢复数据请保留** |
| `mysql/server_asset_backup.sql` | 同上，可能是重复备份 |
| `mysql/` 文件夹 | 若仅用于存放备份，可整文件夹删除 |
| `prisma/` 根目录 | 空文件夹，项目实际使用 `backend/prisma/`，可删除 |

---

## 三、不要删除

| 路径 | 说明 |
|------|------|
| `.env` | 本地环境变量，**不要上传到服务器**，在服务器上单独创建 |
| `backend/.env` | 同上 |
| `backend/scripts/seed-accounts.config.json` | 含 API Token，**不要上传**，在服务器上单独配置 |

---

## 四、PowerShell 一键删除命令（仅删除第一类）

在项目根目录执行（**请先确认路径正确**）：

```powershell
Remove-Item -Recurse -Force "backend\node_modules" -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force "backend\dist" -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force "frontend\node_modules" -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force "frontend\dist" -ErrorAction SilentlyContinue
Write-Host "已删除 node_modules 和 dist，可重新 npm install 恢复"
```

---

## 五、删除后恢复（本地开发时）

```bash
cd backend && npm install && cd ..
cd frontend && npm install && cd ..
cd backend && npm run build && cd ..
```
